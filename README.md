# JTBD Skills

> 基于 JTBD（Jobs To Be Done）框架的一组研究技能，覆盖 **访谈提纲设计** 和 **访谈材料分析** 两个核心环节。

---

## 这是什么

这个仓库把两个互相关联的 JTBD skill 放在一起，方便统一管理、统一分享，也更符合真实研究流程。

它不是单一功能的 skill，而是一套配套工具：

1. **jtbd-interview-guide**，用于设计 JTBD 访谈提纲
2. **jtbd-analyzer**，用于分析访谈文本、访谈笔记和用户反馈

你可以把它理解成一套从“怎么问”到“怎么分析”的 JTBD 研究小工具箱。

---

## 包含的 skill

### 1）jtbd-interview-guide

用于生成 JTBD 访谈提纲，帮助研究者围绕真实事件而不是抽象态度提问。

适合处理：
- 设计用户访谈提纲
- 研究用户为什么选择某个产品 / 服务
- 研究用户为什么切换、放弃或流失
- 挖掘 Trigger、竞争方案和 switch 逻辑

### 2）jtbd-analyzer

用于分析访谈逐字稿、访谈笔记和用户反馈，把原始材料转成结构化 JTBD 洞察。

适合处理：
- 提取 Job Statements
- 构建情境卡片
- 做竞争方案分析
- 做四力分析（推力 / 拉力 / 焦虑 / 惯性）
- 输出洞察与设计建议

---

## 适合的使用顺序

如果你在做一轮完整的 JTBD 研究，通常可以这样用：

### 第一步：先用 `jtbd-interview-guide`
明确研究对象、目标用户和研究目标，生成一套高质量访谈提纲。

### 第二步：完成访谈
获得逐字稿、笔记或整理后的研究材料。

### 第三步：再用 `jtbd-analyzer`
把访谈材料转成结构化分析报告，提取核心 job、情境、触发因素和设计建议。

也就是说，这两个 skill 分别对应研究流程的前后两段：

- **前段**：提纲设计
- **后段**：材料分析

## 什么时候用哪个

| 任务 | 推荐 skill |
|---|---|
| 我要做访谈提纲 | `jtbd-interview-guide` |
| 我要研究用户为什么选择 / 放弃某方案 | `jtbd-interview-guide` |
| 我已经有访谈文本，想提炼洞察 | `jtbd-analyzer` |
| 我想从原始材料中提取 job 和设计建议 | `jtbd-analyzer` |

---

## 使用方式

把对应目录下的 `SKILL.md` 作为主技能提示词加载即可。

如果你是在做完整研究，建议把这两个 skill 组合使用：先设计提纲，再分析材料。
